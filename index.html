Learning client side

I'm a server side c# developer. I'm trying to learn client side using Angular2 and related technologies. Angualar2 is a technology that will allow me to update client side without reloading pages. When I read books and articles about Angular2 they mostly seem to be written for the perspective of a client side developer. They seem to require me to download lots of unfamiliar code and applications and then my web page magically updates. Then, when I change something, it stops working and I don't know what I did wrong. This blog is my journey through picking apart the pieces that make up the Angular2 applications and understanding and documenting what each one does. I'm going to try and build an Angular2 based application from the bottom up

Typescript

Typescript is a language that allows me to create JavaScript code in a type safe way using OOP style classes. The Typescript transpiler (similar to a compiler) then converts this into nasty old JavaScript without me having to get my hands dirty. I don't have to use Transcript to get Angular2 working but it should make my code more reliable. Typescript was developed by Microsoft C# people and it is Open Source and on GitHub, Angular2 is written using Typescript. It does the job and looks like will be around for a while so I think it is worth using. This is the download page for TypeScript. https://www.typescriptlang.org/index.html#download-link. It talks about node.js packages and npm. This was my first hurdle...

Typings

I'm also seeing a typings.json file in some projects. Turns out that many Javascript libraries eg Jquery have extended features beyond Javascript that TypeScript , beacuase these features are not n pukka JavaScrupt , cannot trnaspile. To get around this these libraties often offer a TypeScript definition file that shows Transcript Compiler how to convert them. The typings.json is a standard file that tells tsc where to find the TypeScript definition files. We may as well included this in our folder - don't need it yet but it won't do any harm To actually get the code for these I need to run commands like 'npm install @types/jasmine --save-dev'

node.js and npm

Node.js seems to be the JavaScript equivalent of asp.net. - it generates server side JavaScript files. npm is the node.js equivalent of NuGet. Since Typescript is a JavaScript 'package' I need to download both node.js and npm to get TypeScript. TypeScript is installed as part of some versions of Visual studio and Visual Studio Code. I tried that route and found it pushed me into project templates containing unfamiliar files. Since I want to understand things from the bottom up I downloaded node.js from https://nodejs.org/en/download/ I clicked the Current Latest Features button and downloaded and installed the Windows msi file. This installed both node.js and npm package manager.I install latest npn using. This link https://docs.npmjs.com/getting-started/installing-node tells me that npm gets updated more frequently than nond.js and recommends that i run 'npm install npm@latest -g' and then check the version with npm -v. (Check node version with 'node -v') My versions are: npm : 4.5.0 node : 7.9.0 Now I opened up Command Prompt and pasted in 'npm install -g typescript' from the TypeScript download link.

Did Typescript install OK?

Easiest way to check is to create a simple TypeScript file and then run the Typescript transpiler. Create the simple Typescript file in https://www.typescriptlang.org/docs/tutorial.html and save it as say test.ts. It has to have a ts extension In Command Prompt, navigate to the folder that contains your the file and enter tsc test.ts. You should see a javascript file that tsc has created

Running Transcript Transpiler in Watch mode (-w option)

I found this worked better when I used Cmder Console window(http://cmder.net/ ) instead of Command Prompt. Try entering 'tsc test.tc -w' and then change and save test.ts. The transpiler will automatically detect changes and update the javascript test.js file So I now understand how to automatically update the JavaScript file every time I modify a Transcript file. I still need Angular2 to automatically update the web page though I want to be able to automatically update all the JavaScript files in my project. To do this I need to use the project (-p) option and use a tsconfg.json file. Sample ones are here. https://www.typescriptlang.org/docs/handbook/tsconfig-json.html I modified one of the sample ones to have strict compilation options and to output into separate JavaScript files with no map file. This is what I came up with { "compilerOptions": { "module": "commonjs", "target": "es5", "noImplicitAny": true, "removeComments": true, "preserveConstEnums": true, "alwaysStrict" : true, "diagnostics" : true, "experimentalDecorators" : true, "forceConsistentCasingInFileNames" : true, "noUnusedLocals" :true, "noUnusedParameters" :true, "strictNullChecks" :true, "watch" : true, "pretty" : true, "listEmittedFiles" : true // "outFile": "/output/tsc.js", // "sourceMap": true }, "include": [ "app/**/*" ], "exclude": [ "node_modules", "**/*.spec.ts" ] } module option - TypeScript scopes variables, functions, classes by module - for something to be visible elsewhere it has to be exported. settign module to be commonjs tells the transpiler that I want t use the node.js module loader to import/export my modules - no idea what effect this has I created a second Typescript file. Angular2

packages.json

I now need to add some Angular2 stuff. I need a packages.json file. This file contains several sections described in detail in https://angular.io/docs/ts/latest/guide/npm-packages.html The sections are:

name

The name of my application

version

The version of my software. I'll leave this as 1.0.0 for now

license

What licence I am releasing my application under. I'll leave this as MIT

dependencies and devdependencies

These are similar to my familiar C# packages.config file. Splitting out the dependencies that are only needed for development and not production is useful. So to not include these in production I just need to add the --production option as below (https://angular.io/docs/ts/latest/guide/npm-packages.html) npm install my-application --production Seems like I can just manually remove them prior to deployment. I'm going to use the ones recommened here https://angular.io/docs/js/latest/quickstart.html#!#package-json

scripts

This is optional. The interesting bit is the "start" key which is used to start up our app. This is what 'npm start' actually runs. I'm going to use the scripts section from https://www.tutorialspoint.com/angular2/angular2_environment.htm which is "scripts": { "start": "concurrent \"npm run tsc:w\" \"npm run lite\" ", "tsc": "tsc", "tsc:w": "tsc -w", "lite": "lite-server", "typings": "typings", "postinstall": "typings install" }, but I'm going to leave out the typings and postinstall KVPs because I don't think I need them http://onehungrymind.com/build-a-simple-website-with-angular-2/
